# 子网计算工具 — 功能说明文档

> 本文档根据当前代码实现整理，描述各模块能力、操作要点与使用限制。  
> 工具为**离线前端计算**为主；部分能力依赖 **Electron 桌面环境**。

---

## 一、产品概述

| 项目 | 说明 |
|------|------|
| 名称 | 子网计算工具（IPv4/IPv6）绿色版 |
| 定位 | IPv4/IPv6 子网与地址相关计算、规划与校验 |
| 数据 | 默认不上传数据；计算在本地完成 |
| 运行形态 | ① 浏览器打开 `index.html`（ES Module）② Electron 桌面应用 |

## 二、界面与通用能力

### 2.1 顶部菜单与免责声明

- **免责声明**：在页眉下方以醒目条带展示，使用工具前请仔细阅读（含合法授权、探测行为与责任说明）。
- **导出 CSV / TXT / Excel**：基于**最近一次带结构化数据的计算记录**导出（无 `outputData` 的记录会提示不可导出）。
- **主题**：自动 / 浅色 / 深色（本地 `localStorage` 记忆）。
- **帮助**：弹窗说明与 FAQ。

> 浅色主题下，顶部「导出 / 主题 / 帮助」按钮采用与功能区 Tab 激活态一致的浅蓝底与蓝色边框样式。  
> **帮助**弹窗在浅色主题下使用白底、深色正文，与页面协调；深色主题下仍为深色卡片样式。

### 2.2 历史记录（底部）

- 自动记录最近 **200 条**计算结果。
- 支持**复用**（切换 Tab 并还原输出）、**单条删除**、**清空**；支持**导出 JSON / 导入 JSON**（与现有记录合并，仍受 200 条上限约束）。
- 大整数等会以可 JSON 序列化形式存储。

### 2.3 标签页顺序（从左到右）

1. IPv4 计算  
2. IPv6 计算  
3. **IPv6 子网规划**（VLSM / 等长）  
4. **群 Ping 检测**（仅 Electron）  
5. **端口扫描**（仅 Electron）  
6. **路由追踪**（仅 Electron）  
7. 等长子网规划  
8. 可变长子网规划（VLSM）  
9. CIDR 汇总  
10. 冲突/包含检查  
11. 双栈对比  
12. 地址验证/归属查询  

> **本机网络信息**已移至顶部菜单，在 **「导出 CSV」左侧** 的 **「本机网络」** 按钮，结果在弹窗中展示（仅 Electron）；**10 分钟内**再次打开优先使用**缓存**（点「重新获取」立即拉新）；复制成功为页面**顶部 Toast**（约 1.5 秒消失）。

### 2.4 全局快捷键（全模块）

- **Ctrl+Enter**：在当前标签执行「主操作」（如 IPv4/IPv6 一键计算、群 Ping、端口扫描、路由追踪开始、VLSM 生成等）；焦点在普通输入框/文本域内时不触发。
- **Ctrl+S**：弹出**导出格式三选一**（CSV / TXT / Excel），与顶部三个导出菜单一致，均导出**当前选中的历史记录**。
- **Ctrl+F**：打开**全局搜索**，可跳转到各标签页或从历史记录标题打开对应结果；**Esc** 关闭。

## 三、各功能模块说明

### 3.1 IPv4 核心计算

**能力：**

- 四段式 IP + **CIDR 前缀**与**点分掩码**双向联动；前缀滑块、常用前缀下拉；对非法段值、非连续掩码等提供**智能纠错提示**（见帮助内「智能纠错说明」）。
- 各功能页双栏卡片之间可**拖拽竖条**调整左右宽度（窄屏下自动改为上下布局）。
- 按主机数推荐掩码提示（可选）。
- 可选：**扣除网络地址与广播地址**（常规主机数口径）。
- 输出：网络地址、掩码/CIDR、广播、首末可用主机、可用主机数、子网位/主机位、地址类别与类型、利用率等。
- **反向计算**（同页）：  
  - 模式一：按期望主机数推算前缀与子网参数。  
  - 模式二：按期望子网数推算划分方案预览。  
  - 多种策略：最优划分、不超网、不浪费等。

**操作：** 填写或联动生成 `IP/前缀` → **一键计算** / **开始反向推算**。

---

### 3.2 IPv6 基础计算

**能力：**

- 输入 IPv6 地址 + 前缀长度；前缀数值框、滑块、常用前缀下拉联动。
- 可选展开为 8 组显示。
- 输出：展开/压缩形式、网络前缀、接口 ID、地址类型、前缀起止范围等。

---

### 3.2.1 IPv6 子网规划（VLSM / 等长）

**能力：**

- **父网段**：输入 IPv6 CIDR（如 `2001:db8::/48`）。
- **VLSM（按地址需求）**：多条需求，每条填写「所需地址数量」（按 2 的幂对齐的地址块，思路类比 IPv4 主机数）；策略包括最大/最小优先、固定掩码、按需匹配、顺序、平衡等（与 IPv4 VLSM 命名一致）。
- 输出：各子网 **CIDR**、**首末地址**、块内地址数、**主机位（接口 ID 位数）** 及 **/64 与接口 ID 的常见规划提示**。
- **等长划分**：在同一父网段上指定子网个数，自动加长前缀并列出每个子网的网络前缀与地址范围（可选展开 8 组显示）。

---

### 3.3 群 Ping 检测（Electron 专用）

**依赖：** `main.js` 通过 `preload.js` 暴露 `window.subnetNative.batchPing`，调用系统 `ping` 命令。

**输入：**

- **IPv4 四段式 + 掩码双显联动**，组合为 CIDR；对网段内地址逐 IP 检测（单网段最多 **4096** 个地址，超出会报错提示缩小网段）。

**参数：**

- **超时（毫秒）**：约 200～10000。

**结果展示：**

- 表格：序号、IP、状态、耗时(ms)。
- **可达**：IP 与状态为**绿色**；**不可达**：**红色**。
- 标题行右侧 **统计摘要**：总计、可达数、不可达数、当前筛选下的显示条数。
- **状态筛选**（位于右侧「网段延迟分布」图例旁）：**全部 / 仅可达 / 仅不可达**；与表格一致，**色块网格也会按同一规则只显示符合筛选的地址格**（未通过筛选的格不渲染，网格会相应变稀疏）。
- **分页**：每页 **10 / 20 / 50**，上一页 / 下一页，页码信息；筛选变更时回到第 1 页。
- **色块可视化（网段延迟分布）**：
  - **随 IPv4 网段输入即时预览**：在左侧四段式 + 掩码联动生成有效 CIDR 后，无需等待 Ping 即可在右侧显示色块网格；未检测的主机格为**待检测**样式，Ping 完成后在同一网格上按延迟着色。
  - 在明细表上方展示 **15 列** 网格，按网段内地址顺序为每个地址显示一格。
  - 图例：**&lt;50ms**（绿）、**50–100ms**（黄）、**&gt;100ms**（红）、**超时/不可达**（灰）、**网络地址**（橙）、**广播地址**（紫）。
  - **显示网络地址 / 显示广播地址**（复选框，位于「网段延迟分布（色块）」标题同一行右侧，**默认不勾选**）：未勾选时，色块网格与明细列表中均**不展示**该网段的网络地址与广播地址；勾选后显示，且色块中网络地址为**橙色**、广播地址为**紫色**（与延迟色独立；若同时勾选且该地址有 Ping 结果，提示里仍会带延迟说明）。
  - 若所有地址同属一个 **/24** 子网，格内数字为 **末段主机号**（如 `1`～`254`）；否则格内显示完整 IP（字体会自动换行以适配小格）。
  - 延迟优先取系统 `ping` 输出中解析的 **ICMP 往返时间**（`rttMs`）；解析失败时可用总耗时近似，色块分档仍以该数值为准。

**导出：**

- **导出 Ping Excel**：生成 `.xls`（HTML 表格方式，Excel 可打开）。

**其它：**

- 检测**结束**（成功或失败）后在**屏幕右下角**弹出任务通知卡片，同时主窗口**任务栏按钮闪烁**提示；**点击通知**可唤起主窗口并自动切换到「群 Ping」页（点 × 仅关闭通知并停止闪烁）。不再使用声音提示。

**限制：**

- 浏览器直接打开 `index.html` 时**无** `subnetNative`，群 Ping 不可用（会提示不支持系统 Ping）。
- 实际能否 ping 通受本机网络、防火墙、目标是否禁 ICMP 等影响。

### 3.3.1 端口扫描（Electron 专用）

**依赖：** `main.js` 通过 `preload.js` 暴露 `window.subnetNative.tcpPortScan`（Node `net` 单次 TCP 连接探测）；可选调用 `batchPing` 做 ICMP 预筛。

**输入：**

- 与群 Ping **相同**的 **IPv4 四段式 + 掩码双显联动**，生成 CIDR；单网段最多 **4096** 个地址（与群 Ping 一致）。
- **端口列表**：逗号/空格分隔，**最多 32 个**（1–65535）。
- **TCP 连接超时**（约 200～8000 ms）。
- **并发 TCP 数**（默认 8，上限 32，建议轻量 4～16）。
- **任务间隔**（毫秒，0 表示不延迟）：每个探测任务开始前可插入短延迟，降低瞬时连接峰值。
- **先 Ping 再扫**（可选）：先对网段内地址做 ICMP Ping，**仅对可达主机**发起 TCP 探测；使用单独的 **ICMP 超时** 设置。
- **跳过网络地址与广播地址**（默认勾选）：与常规主机扫描口径一致。

**探测规模：**

- 总任务数 =（参与扫描的 IP 数）×（端口数），上限 **25000**；超出需缩小网段或减少端口。

**结果：**

- 表格：**IP、端口、状态（开放/关闭）、耗时(ms)、备注**（如超时、连接被拒等错误码摘要）。
- **状态筛选**：全部 / 仅开放 / 仅关闭；**分页** 10 / 20 / 50。
- **导出扫描 Excel**：生成 `.xls`（HTML 表格）；历史记录亦支持顶栏 CSV/TXT/Excel 导出。

**说明（轻量与安全）：**

- 仅尝试 **TCP 三次握手**，连接建立后立即关闭，**不发送应用层数据**；请在**已获授权**的网络内使用。
- 目标或中间设备仍可能记录连接尝试；若需更低存在感，请减小并发、增大间隔、缩小网段。

**限制：**

- 浏览器直接打开 `index.html` 时无 `subnetNative`，端口扫描不可用。

**其它：**

- 扫描**结束**（成功或失败）后的提示方式与群 Ping **一致**（右下角通知 + 任务栏闪烁 + 点击回到「端口扫描」页）。

### 3.3.2 路由追踪（Electron 专用）

**依赖：** `preload.js` 暴露 `window.subnetNative.traceRoute`、`traceRouteAbort`、`traceRoutePauseToggle`；主进程调用系统 **`tracert`（Windows）** 或 **`traceroute`（Linux/macOS）**，解析标准输出。

**输入：**

- **目标**：IPv4、IPv6 或域名（会做基本合法性校验，禁止换行与引号等字符）。
- **最大跃点数**：1～128（默认 30）。
- **单跳等待超时（毫秒）**：对应 Windows `tracert -w`；类 Unix 下会换算为秒级等待参数。
- **停止**：终止当前 `tracert`/`traceroute` 子进程，保留已输出部分用于解析与展示。
- **暂停 / 继续**：在 **macOS / Linux** 上对子进程发送 `SIGSTOP` / `SIGCONT`（**Windows 不支持**，按钮会提示）。

**结果：**

- **逐跳摘要表**：跳序号、**IP**、节点原文、**平均延迟**（由每跳 3 次探测推算）、**丢包率**（超时次数 / 3）、**三次探测延迟**（`*` 表示该次超时）。
- **原始输出**：完整命令输出便于核对；可 **复制原始输出**。
- **Windows 乱码**：主进程对 `tracert` 输出优先按 **cp936（GBK）** 解码（依赖 `iconv-lite`），减少简体中文环境下的乱码。

**其它：**

- 追踪**结束、手动停止或失败**后的提示与群 Ping **一致**（右下角通知 + 任务栏闪烁 + 点击回到「路由追踪」页）。
- 全程可能持续数分钟；主进程对子进程设有最长等待，超时将终止并返回已捕获输出。

### 3.3.3 本机网络信息（Electron 专用）

**入口：** 顶部菜单 **「本机网络」**（在 **「导出 CSV」** 左侧），点击后在**弹窗**中加载并展示。

**依赖：** `preload.js` 暴露 `window.subnetNative.getLocalNetworkInfo`；主进程使用 Node `os.networkInterfaces()`；在 **Windows** 下通过 **UTF-8（带 BOM）临时 `.ps1` 脚本 + `powershell -File`** 调用 `Get-NetIPConfiguration` / `Get-DnsClientServerAddress`，避免编码错误导致乱码或 CLIXML 混入。

**能力：**

- 展示各网卡 **MAC**、**IPv4/IPv6 地址**、**子网掩码 / CIDR**（以系统与 Node 返回值为准）。
- Windows 下对同名适配器合并展示 **网关 IPv4/IPv6**、**DNS IPv4/IPv6**。
- 弹窗内 **复制**单行、**复制本卡全部**；底部 **复制全部** 导出纯文本；复制成功后在页面**最上层 Toast 轻提示**（失败亦为 Toast）。

**限制：**

- 非 Windows 平台可能仅有 `os.networkInterfaces()` 信息，网关/DNS 需自行用系统命令查看。
- 浏览器环境无此 API，将提示使用 Electron。

### 3.4 等长子网划分

**能力：**

- 父网段（四段 + 掩码联动）+ 子网数量 + 起始编号。
- 可选扣除网络/广播以显示可用主机范围。
- **划分预览**（仅提示推荐前缀与数量）；**生成所有子网**（完整表格：编号、网段、掩码、建议/备选网关、范围、广播、可用主机数）。

### 3.5 VLSM（可变长子网掩码）

**能力：**

- 父网段 + 多条「子网名称 + 需求主机数」需求表（增删改）。
- 多种分配策略（最大优先、最小优先、固定掩码、Best Fit、顺序、平衡、聚合导向、碎片整理等）。
- 输出拓扑清单与汇总（利用率、剩余地址等）。

---

### 3.6 CIDR 汇总（超网）

**能力：**

- 批量输入 IPv4 CIDR，在**不引入超出输入并集的空洞**的前提下做严格可合并汇总。
- 展示汇总前后列表与减少的路由条目数说明。

---

### 3.7 冲突/包含检查

**能力：**

- 支持 IPv4 / IPv6 混输（CIDR 或单地址按 /32、/128 处理）。
- 输出两两关系：完全重叠、部分重叠、包含等。
- IPv4 附带简易条带可视化（区间相对位置示意）。

---

### 3.8 双栈对比

**能力：**

- 左侧 IPv4（四段 + 掩码联动）；右侧 IPv6 需 `地址/前缀` 格式。
- 可选 IPv4 是否扣除网络/广播。
- 同屏展示 IPv4 核心结果与 IPv6 核心字段对比。

---

### 3.9 地址验证与归属查询（离线）

**能力：**

- 四段式 IPv4：**判断地址类型**（公网/私网、A/B/C/D/E、环回、链路本地等）。
- **可选**加载离线 JSON 归属库（前缀列表），对公网地址做前缀最长匹配，显示组织/地域/ASN 等。
- 默认内置 `ip_geo_db.js`（可为空数组，不影响类型判断）。

## 五、构建与分发（参考）

## 六、已知限制摘要

1. **群 Ping** 仅 Electron；网页版无系统 `ping` 能力。  
2. **单网段 Ping** 地址数上限 **4096**，防止界面与系统负载过大。  
3. **Excel 导出** 为 HTML-Excel（`.xls`），非原生 xlsx。  
4. **历史/项目导出** 依赖各次计算是否写入 `outputData`；部分类型仅 TXT/部分字段为空。  
5. 子网与地址计算逻辑以 `netcalc.js` 实现为准；极端输入以界面报错信息为准。
